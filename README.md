# Baby Name Explorer

A React app that lets expecting parents explore the popularity and gender
fixedness of any American baby name using SSA birth records from 1880–2024.

Built for the Gender and Baby Names Storytelling Project.

---

## Project structure

```
baby-name-explorer/
  scripts/
    process_ssa.py       # data pipeline: SSA → static JSON files
  frontend/
    public/
      data/              # generated by process_ssa.py (you populate this)
        index.json
        names/
          Jordan.json
          Sophia.json
          ...
    src/
      App.jsx            # main React component
      App.css
      index.js
    package.json
```

---

## Step 1 — Generate the data

You need the raw SSA files OR your compiled `ssa_names_compiled.csv` (the one
from your existing notebook workflow).

```bash
pip install pandas numpy

# Using your compiled CSV (recommended — matches your notebook exactly):
python scripts/process_ssa.py \
  --compiled /path/to/ssa_names_compiled.csv \
  --out-dir frontend/public/data

# Or from raw SSA yearly .txt files:
python scripts/process_ssa.py \
  --ssa-dir /path/to/names/ \
  --out-dir frontend/public/data
```

This writes ~51k JSON files into `frontend/public/data/names/` and an
`index.json`. Expect it to take a few minutes. ~150–250 MB total.

**Fixedness formula** is identical to `gender_fixedness_score_redesign.ipynb`:
- Smoothed p: `(F + α) / (F + M + 2α)` with α=10
- Confidence: `n / (n + k)` with k=200
- Fixedness: `signal × confidence`
- Temporal fixedness: applies cross-decade stability penalty

## Step 2 — Run locally

```bash
cd frontend
npm install
npm start
```

The app fetches JSON files from `./data/` in `public/`. No backend server needed.

## Step 3 — Deploy to GitHub Pages

### 3a. Create a GitHub repo

```bash
git init
git add .
git commit -m "initial commit"
gh repo create baby-name-explorer --public
git push -u origin main
```

### 3b. Update homepage in package.json

Edit `frontend/package.json`:

```json
"homepage": "https://YOUR-USERNAME.github.io/baby-name-explorer"
```

### 3c. Build and deploy

```bash
cd frontend
npm run build
npm run deploy
```

This publishes the `build/` folder (including your `data/` files) to the
`gh-pages` branch. GitHub Pages serves it automatically.

> ⚠️ **File size note**: ~51k JSON files + index.json is large for a git repo
> (~150–250 MB). If you hit GitHub's 1 GB limit or slow deploy times,
> consider filtering to names with ≥100 or ≥250 all-time births
> (`--min-births 250`) which drops the set to ~26k names while still covering
> 99%+ of births.

## Claude API enrichment

The app calls the Anthropic API (Claude) for name meaning, origin, and usage
context. This requires the API to be accessible from the user's browser.

**For local dev**: works out of the box.

**For production**: you'll want a small proxy (e.g. a Cloudflare Worker or
Vercel Edge Function) that injects your API key server-side rather than
exposing it in the browser. The app degrades gracefully if enrichment fails —
it just hides the meaning card.

## Customizing

- **Year defaults**: change `yearStart` / `yearEnd` useState values in `App.jsx`
- **Fixedness thresholds**: `getFixednessLabel()` in `App.jsx`
- **Min births**: `--min-births` flag to `process_ssa.py` (default 50)
- **Data URL**: set `REACT_APP_DATA_URL` in `.env.local` to point at a CDN

---

## Data source

US Social Security Administration: https://www.ssa.gov/oact/babynames/limits.html

SSA suppresses name-year-sex combinations with fewer than 5 births. These
appear as 0 in the output.
